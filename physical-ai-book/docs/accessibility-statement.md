# Accessibility Statement

## Commitment to Accessibility

The Physical AI & Humanoid Robotics Educational Book is committed to ensuring digital accessibility for all users, including those with disabilities. We are continuously improving the user experience for everyone and applying the relevant accessibility standards.

## Conformance Status

This website follows the Web Content Accessibility Guidelines (WCAG) 2.1 AA standards. These guidelines explain how to make web content more accessible for people with disabilities and more user-friendly for everyone.

## Accessibility Features

### Keyboard Navigation
- All interactive elements are accessible via keyboard
- Logical tab order throughout the site
- Visible focus indicators for keyboard users
- Skip-to-content links for faster navigation

### Screen Reader Compatibility
- Proper heading hierarchy (H1, H2, H3, etc.)
- Descriptive alt text for all meaningful images
- ARIA labels for complex interactive elements
- Semantic HTML structure

### Visual Accessibility
- Sufficient color contrast (at least 4.5:1 for normal text, 3:1 for large text)
- Text scaling up to 200% without loss of functionality
- Responsive design that works across devices
- Dark mode support for reduced eye strain

### Content Accessibility
- Clear, simple language appropriate for the target audience
- Consistent navigation and page structure
- Descriptive link text that indicates the link's purpose
- Proper use of lists, tables, and other structural elements

## Ongoing Efforts

We are actively working to maintain and improve accessibility through:

1. **Regular Audits**: Using automated tools and manual testing
2. **User Feedback**: Encouraging reports of accessibility issues
3. **Content Creation Standards**: Ensuring all new content meets accessibility requirements
4. **Training**: Educating content creators on accessibility best practices

## Known Limitations

While we strive for full compliance, we acknowledge some limitations:

- Some diagrams and complex visualizations may require additional descriptive text
- Code examples may need improved color contrast for syntax highlighting
- Third-party embedded content may not meet all accessibility standards

## Feedback and Support

We welcome feedback on the accessibility of this website. If you encounter accessibility barriers or have suggestions for improvement, please contact us through our GitHub repository.

### Reporting Issues
- **What to include**: Description of the problem, URL where it occurs, assistive technology used, and suggested solution
- **How to report**: Create an issue in our GitHub repository with the "accessibility" tag
